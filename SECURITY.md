# Security Policy

## Supported Versions

PureLink is pre-1.0 software. Security fixes are applied to the default branch until versioned releases begin.

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues.

To report a vulnerability, use GitHub private vulnerability reporting for `github.com/MasuRii/PureLink` if it is enabled, or open a private security advisory from the repository's **Security** tab.

Include as much detail as possible:

- Affected PureLink version or commit SHA
- Operating system and architecture
- Steps to reproduce
- Expected and observed behavior
- Any proof-of-concept input or endpoint data that is safe to share

Maintainers will acknowledge valid reports as soon as practical and coordinate a fix before public disclosure.
